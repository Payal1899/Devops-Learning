Install docker need apt
cmd: apt install docker.io
docker --version
if sudo permission needs>>post installation steps docker will help to remove need of sudo

Docker Pull:
$docker pull image_name
~downloads the container for that image_name

Docker images:
$docker images
~shows images downloaded on local system

Docker run:
$docker run image_name
~runs image_name

$docker run -it image_name
~runs image_name and interactive + attach to terminal, it takes us to container id in terminal
now we are on container id.
many cmds dont work inside this as it loads basic image and not all the utilities
from this terminal at container ID(which means we are inside VM), run #apt update

Docker ps:
$docker ps image_name
~lists running image_name

-a options shows: past runs, stopped images

i)docker ps -aq
ii)docker rm $(docker ps -aq)
check stopped imges, q means only container_id
then remove all those container_id

docker image prune
WARNING! This will remove all dangling images, images which were build previously and having tag none also none id

Docker stop:
$docker stop container_id
~stops running container_id

Docker start:
$docker start container_id

***************************
accesing docker container with port mapping

docker run -it -p 7790:80 nginx
docker run -it -p internal(Local)_port:external_port nginx
7790 = port on your computer (host)
80 = port inside the container (NGINX runs here)
👉 So visiting http://localhost:7790 opens the NGINX server.
nginx
The Docker image to run (a web server).

**********************
docker logs container_id
shows logs for container id

docker exec -it container_id command_to_run 
This allows to run command from outside to docker container
so basically you mapped your local machine port to docker container port and from your local machine terminal now executing command inside docker container terminal

ex : docker exec -it container_id cat names.txt
it shows names.txt file content from that container_id


Docker inspect:
docker inspect container_id
gives info about container

Creating Docker image

1)Docker file
blueprint for container
create docker file, give lib, file and filepath, and command to run


in terminal> docker build -t python-first-image .
docker build>builds image
-t> tag followed by image name, tag is version of image by default it is 'latest'
.>current dir(it should be dir where dockerfile exist)
~hence docker image created

docker run python-first-image

docker run python-first-image:V2
